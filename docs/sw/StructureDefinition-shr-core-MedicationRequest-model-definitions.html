MedicationRequest

Definitions for shr-core-MedicationRequest-model.

shr-core-MedicationRequest-model
Definition

An order for a medication to be dispensed and instructions for use. Value sets for status are in direct conflict between DSTU2 Argonaut and STU3 US-Core. Category does not exist in DSTU2 and is provided as an extension. Intent was added in STU3 and is provided as an extension with required binding.

shr-core-MedicationRequest-model.numberOfRefillsAllowed
Definition

The maximum number of times the dispense can be repeated. For medication dispense, this integer does NOT include the original order dispense. This means that if an order indicates dispense 30 tablets plus 3 repeats, then the order can be dispensed a total of 4 times and the patient can receive a total of 120 tablets.
